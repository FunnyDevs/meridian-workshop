Known Gaps & Open Items

⚠️
Backlog view has no route
Backlog.vue exists but is not registered in the router (main.js). The view is unreachable from the application.
⚠️
No real database
All data lives in JSON files loaded in-memory at startup. Changes (e.g. new purchase orders) are lost on server restart. Not suitable for multi-user or production use without a proper database layer.
⚠️
Tasks and Purchase Orders are stubs
POST/DELETE/PATCH endpoints for tasks and purchase orders exist in the API client but backend logic is incomplete. Data does not persist.
⚠️
Demand and Backlog filters are client-side only
The /api/demand and /api/backlog endpoints return all data without filtering. Warehouse/category filtering happens in the frontend by matching SKUs against inventory — inconsistent with how other views work.
⚠️
No authentication
User session is mocked in useAuth.js. CORS allows all origins. Suitable for internal demo use only — not production-ready.
ℹ️
Options API remnants
Some views were partially migrated from Vue 2 Options API to Vue 3 Composition API. Migration is incomplete — inconsistent patterns remain in the codebase.
